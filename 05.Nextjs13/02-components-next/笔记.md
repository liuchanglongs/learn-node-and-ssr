# 内置组件

Next.js 框架也提供了几个内置组件，常用的有：

1. Head：用于将`新增的标签`添加到页面的 `head` 标签中(就是 html 中的 head 下面的标签)，需要从 next/head 中导入

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="shortcut icon" href="favicon.ico" type="image/x-icon" />
  </head>

  <body></body>
</html>
```

- 如果想要给`页面`单独添加 head 的一些设置， \_document.js 文件中必须要使用 Head,不然不会生效

```html
 <Html lang="en">
      {/* 这里的SEO的配置,是作用域所有的页面 */}
      <Head>
        {/* <title>我是Title 111</title>
        <meta name="description" content="网易云音乐商城"></meta>
        <link rel="shortcut icon" href="favicon.ico" type="image/x-icon" /> */}
      </Head>
      {/* <Script src="http://localhost:3000"></Script> */}
      <body className="lcl">
        <Main />
        <NextScript />
      </body>
    </Html>
```

- 如果想要给`所有页面`统一添加的，那需在 pages 目录下新建 \_document.js 文件来定制 HTML 页面

2. Script（是对 script 标签 的增强）：**将一个 script 标签到页面的 body 中（不支持在\_document.js 中用）**，需要从 next/script 中导入。

- 官网链接：https://nextjs.org/docs/app/api-reference/components/script

3. Link：可以启用客户端的路由切换，需从 next/link 导入
4. Image：内置的图片组件（对 img 的增强）。需从 next/image 导入

- Image 组件常用属性
   src 属性：
  ✓ **引入本地图片资源，会自动确认图片的宽高**，
  ✓ 引入外部资源需，`需手动给宽高，还需配置白名单`。
   width/height：是 number 类型，不支持 100%等字符串
   priority：将图片标记为 LCP ( Largest Contentful Paint )元素，允许预加载图像。

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "**.music.126.net",
      },
    ],
  },
};

module.exports = nextConfig;
```

✓ 建议大图，并在首屏可见时才应使用该属性。默认为 false
 placeholder：图片占位，默认值为 empty，当值为 blur 需和 blurDataURL 一起用
 fill ：让图片填充父容器大小，父容器需设为相对定位

- 官网链接：https://nextjs.org/docs/app/api-reference/components/script
