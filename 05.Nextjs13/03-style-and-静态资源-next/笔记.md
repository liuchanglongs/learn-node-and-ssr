# 全局和局部样式

1. 内置 Scss 支持: 用 scss 之前，需安装 Sass： npm i sass –D
2. Next.js 允许在 JavaScript 文件中直接通过 import 关键字来导入 CSS 文件（不是：@import ）
3. 全局样式：`在 assets 目录或 styles 目录`下编写，`然后在 pages/_app.js` 入口组件中导入
4. 局部样式：支持导入 node_modules 中样式(跟单页面的 react 一样)，`导入文件后缀名不能省略

- Next.js 默认是支持 CSS Module 的，如：[name].module.css
- CSS Module 中的选择器会自动创建一个唯一的类名。
- 唯一类名保证在不同的文件中使用相同 CSS 类名，也不用担心冲突
- `scss 导出的变量:xx.module.scss文件:export中定义的变量，可导出供JavaScript中用`

```scss
// 导出了一个 scss 变量, 是专门给js代码使用 ( 必须要放在css module的文件中 )
:export {
  // primaryColor: red;
  // primaryColor: $fsColor;
  primaryColor: $primaryColor;
}
```

5. scss 的变量: [代码](styles/main.scss)

- scss 引入 scss:两种方式

```scss
// @use "./variables.scss" as *;
@import "./variables.scss";
```

# 静态资源引用

1. public 目录
    常用于存放静态文件，例如：robots.txt、favicon.ico、img 等，并直接对外提供访问。
    访问需以 / 作为开始路径，例如：添加了一张图片到 public/me.png 中
   ✓ `可通过静态URL`： /me.png 访问，如右图
   ✓ `静态URL也支持在背景中使用`
    `注意： 确保静态文件中没有与 pages/ 下的文件重名，否则导致错误`。

```js
  <Image src={"/feel.png"} alt="feel" width={140} height={140}></Image>
   <div className="box1-bg"></div>
```

```css
.box1-bg {
  width: 140px;
  height: 140px;
  border: 1px solid red;
  background-image: url(/feel.png);
}
```

2. assets 目录
    常用存放样式、字体 、图片或 SVG 等文件
    可用 import 导入 位于 assets 目录的文件，`支持相对路径和绝对路径`
   ✓ import Avatar from “../assets/images/avatar.png”
   ✓ import Avatar from “@/assets/images/avatar.png”
    背景图片和字体： url("~/assets/images/bym.png")

```css
.box2-bg {
  width: 200px;
  height: 200px;
  border: 1px solid red;
  background-image: url(~/assets/images/user.png);
}
```

# 字体图标使用

1. 将字体图标存放在 assets 目录下：

- 调整使用的字体资源@font-face->src：[text](assets/cus-font/iconfont.css)

```css
@font-face {
  font-family: "iconfont"; /* Project id  */
  src: url("~/assets/cus-font/iconfont.ttf") format("truetype");
}
```

2. 字体文件可以使用相对路径和绝对路径引用。
3. 在\_app.tsx 文件中导入全局样式
4. 在页面中就可以使用字体图标了: className="iconfont icon-bianji"

```js
<i
  className="iconfont icon-bianji"
  style={{
    fontSize: "40px",
    color: "#096",
  }}
></i>
```
