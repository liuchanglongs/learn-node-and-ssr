### SSR的问题（pageRouter）

1. 等数据加载完毕后，整个页面才会渲染。
2. JS需要会被打包到前端，增加JS包大小:例如getServerSideProps 返回的 props 会塞进页面里的 **NEXT_DATA**，数据越大，HTML/内联 JSON 也越大。
   ![alt text](https://raw.githubusercontent.com/lcllpz/storage-imgs/main/nuxtr/1735521764762-b0e7a1df-45de-414c-b949-6f284c1dcd021.png)
3. Hydration，整页 React 组件逻辑仍会打进浏览器要下载的 JS。在page Router不管是ssr、ssg，都有Hydrate 。
   > 先服务端出 HTML + 再 hydrate

### RSC服务器组件

<font style="color:rgb(23, 23, 23);">React Server Components</font>也是服务端渲染的，是基于SSR之上的方案。

在/app下创建的页面，默认就是服务器组件。

优势：

- 预渲染、流式处理（Suspense + lazy）
- 减少所需的客户端 JavaScript 数量
- <font style="color:rgb(23, 23, 23);">...</font>
- csr才会发生Hydration
