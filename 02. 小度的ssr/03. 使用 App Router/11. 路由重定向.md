### redirect重定向

临时重定向（307）：`redirect()`  
永久重定向（308）：`permanentRedirect()`（二者常见用在服务端组件、Server Action、Route Handler 等可执行服务端逻辑的场景；客户端程序化导航更常用 `useRouter`。）

```typescript
import { redirect } from "next/navigation";
import { permanentRedirect } from "next/navigation";
```

> 服务端组件重定向会走 HTTP 重定向语义（请求状态默认 307 临时；permanentRedirect 是 请求状态308 永久），浏览器/客户端会按 Location 去加载目标路由（新的导航、新的服务端参与方式，视目标路由而定）。

:::info
场景：

临时的案例：从创建帖子重定向到这个新帖子展示页

永久的案例：从修改个人信息重定向到个人信息展示页

:::

### 配置文件重定向

配置文件next.config.ts中进行重定向设置，可以307也可以308

- CSR 、SSR 都生效

```typescript
async redirects() {
  return [
    {
      source: '/setting',
      destination: '/about',
      permanent: true   // true永久重定向，false临时重定向
    }
  ]
}
```

### 307 与 308 的意义与作用（小结）

- **结论**：307（临时）和 308（永久）不是“随便选的数字”，而是 **HTTP 对这次跳转给出的语义**：爬虫、缓存、客户端是否会 **长期记住新 URL** 会受影响。
- **307 临时**：表示 **这次** 先去 `Location`；**不承诺**旧路径以后永远不用；更像流程跳转（未登录去 `/login`、提交后去结果页等）。搜索引擎通常 **不**把旧 URL 当成“永久作废并合并到新 URL”。
- **308 永久**：表示资源 **永久** 搬到 `Location`，**以后应以新 URL 为准**；更适合路由永久改名、域名/路径规范化。搜索引擎更容易把 **权重/规范 URL** 往新地址 **合并**（具体仍受页面内容与站点策略影响）。
- **和 301/302 的直觉关系**：307/308 都强调 **保留原 HTTP 方法**（例如 POST 仍是 POST），比传统 301/302 在“别把 POST 悄悄变 GET”上更明确。
- **在 Next 里怎么对应**：`redirect()` → **307**；`permanentRedirect()` → **308**；`next.config` 里 `redirects()` 的 `permanent: false/true` 也是同一套 **临时 vs 永久** 语义，只是写在配置里。
- **常见误区**：永久重定向 **不会**清空用户浏览器历史；它主要影响 **后续访问打哪个 URL**、以及 **SEO/缓存** 的长期行为。别把“临时流程跳转”误设成永久，避免收录与外链被错误固化。
- **怎么验证**：DevTools → Network → 看文档请求的 **状态码（307/308）** 与响应头 **`Location`**。

- 使用：
  > 临时 307：if (!user) redirect('/login') —— 不是宣布「全站 /dashboard 永远没了」，只是 这次 先去登录。
  > 永久 308：/blog/post-old-slug 永远改成 /blog/post-new-slug，且希望 外链、收录 慢慢统一到新的。
