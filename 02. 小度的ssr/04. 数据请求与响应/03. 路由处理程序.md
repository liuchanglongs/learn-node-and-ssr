### 概述

路由处理程序允许您使用 Web请求和响应 API 为给定路由创建自定义请求处理程序。

支持以下HTTP 方法：`GET`、`POST、``PUT`、`PATCH、``DELETE、``HEAD` 和`OPTIONS`。如果调用了不受支持的方法，Next.js将返回`405 Method Not Allowed` 响应。

除了支持原生请求和响应 API 之外，Next.js 还通过NextRequest 和NextResponse 对其进行了扩展，以便为高级使用案例提供方便的帮助程序。

> 默认文件读取时route.ts不是index

好处：

- 自定义请求处理
- 简单的mock数据
- 直接调用数据库
- 对请求进行统一拦截

### 参数处理

```shell
// GET DELETE
const searchParams = new URL(request.url).searchParams

// POST PUT
const requestJson = await request.json()
```
