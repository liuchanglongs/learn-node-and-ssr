### 概述

Server Actions（服务器操作：是在服务器上执行的异步函数。可以在 Server 和 Client Components 中调用它们，以处理 Next.js 应用程序中的表单提交和数据更改。

好处：

- 减少fetch和路由处理程序的调用
- 与表单结合非常的灵活

> 提交之后由谁在服务器执行（写库、鉴权、revalidate），不必手写 fetch('/api/...')

> Server Actions 是 Next 提供的 另一条路：把“发请求 + 调服务端函数”合成一步，并且和 RSC、缓存、表单增强绑在一起。

调用服务器函数主要有两种方式：

> https://nextjs.org/docs/app/getting-started/mutating-data

- 服务器和客户端组件中的表单
- 客户端组件中的事件处理程序和useEffect中的逻辑

服务器组件：内联的形式只有服务器组件可以使用

```typescript
export default async function Page() {
  async function createAction() {
    'use server'
    return 'create action'
  }
  return (
    <div>
      ...
    </div>
  )
}
```

客户端组件：要在客户端组件中调用服务器操作，请创建一个新文件并在其顶部添加“use server”`指令。

```typescript
import { createAction } from "./actions";
```

:::info
注意：server action必须是一个async异步函数

:::

### 将操作作为 props 传递

你还可以将服务器操作作为属性传递给客户端组件（没有use server的函数不能传递）：

```typescript
<ClientComponent createAction={createAction} />
```
